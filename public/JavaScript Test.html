JavaScript Test - Creating a simple Path Selector.

The purpose of this test is to build a simple tool that will allow us to extract a CSS selector path of any element within a page.

Part 1 - Adding CSS and HTML to the page dynamically

Write a function that will add some simple CSS into a pages header along with an ID

The CSS to add will be this:

<style id="smc-hover-css">
smc-path-box{
	position:fixed;
	bottom:20px;
	left:10px;
	right:10px;
	padding:3px;
	font-size: 16px;
	background-color: rgba(87, 12, 12, 0.8);
	color: white;
	z-index: 999999999;
	text-align:center;
}
</style>

Also add the following HTML dynamically into the pages <body> tag:

<smc-path-box>Selector Path: </smc-path-box>

Part 2 - Element Hover

Next we want a function that when the mouse hovers over any given element (Ignoring the <body>) it adds the attribute data-smc-hover="true" to the element, and removes the attribute when not hoving over.

For example hovering over a <div> will turn it into <div data-smc-hover="true"> ... </div> and therefore we can then give it a red outline.

Helpful Hint: You can edit your CSS function above to add the red outline via CSS to any element with the data-smc-hover="true" attribute added using [data-smc-hover="true"]{ ... }

Part 3 - Getting the elements path

Now when a highlighted element is clicked we want to try and find it's selector path, or ID if it has one - then output it's value to the <smc-path-box>Selector Path: </smc-path-box>
